NetApp ONTAP Export Policy Guide

Understanding NFS Access Control Using Export Policies

1. What is an Export Policy?

An Export Policy in ONTAP controls which NFS clients can access a volume, what permissions they receive, and which security mechanisms are allowed.

Think of it as a firewall rule for NFS exports.

Key Point:
A volume can be mounted only if the client's IP matches an export-policy rule.

2. Components of an Export Policy Rule

Field Description
Client Match IP address, subnet, hostname, or netgroup
RO Rule Read-only access
RW Rule Read-write access
Protocol NFSv3, NFSv4, NFSv4.1
Superuser Root access mapping

3. Example Environment

Item Value
SVM svm_prod
Volume vol_data
Junction Path /data
Client IP 10.10.10.50

4. Create Export Policy

cluster::> export-policy create \
-vserver svm_prod \
-policyname linux_policy
Verify: 
cluster::> export-policy show

5. Add Export Rule

Allow host 10.10.10.50 read-write access. 
cluster::> export-policy rule create \
-vserver svm_prod \
-policyname linux_policy \
-ruleindex 1 \
-clientmatch 10.10.10.50 \
-protocol nfs3,nfs4 \
-rorule sys \
-rwrule sys \
-superuser sys
Verify: 
cluster::> export-policy rule show \
-vserver svm_prod \
-policyname linux_policy

6. Assign Export Policy to Volume

cluster::> volume modify \
-vserver svm_prod \
-volume vol_data \
-policy linux_policy
Verify: 
cluster::> volume show \
-fields policy

7. Client Side Verification

Check exported paths: 
showmount -e 192.168.1.100
View mount information: 
mount | grep nfs
Mount volume: 
mkdir /mnt/data

mount -t nfs \
192.168.1.100:/data \
/mnt/data
Verify access: 
touch /mnt/data/testfile

echo "Hello ONTAP" > /mnt/data/testfile

ls -l /mnt/data

8. Troubleshooting Commands

Check Export Policy

export-policy rule show

Check Volume Policy

volume show -fields policy

Check NFS Status

vserver nfs show

Check LIFs

network interface show

Check Junction Path

volume show -fields junction-path

9. Common Issues

Issue Possible Cause
Access Denied Client IP not matching export rule
Read Only RW Rule missing
Mount Fails NFS service disabled
No Route LIF unreachable
Permission Denied UNIX permissions issue

10. Export Policy Evaluation Flow

Client Mount Request
          |
          v
Export Policy Assigned?
          |
          v
Rule Match Found?
          |
          v
Protocol Allowed?
          |
          v
RO/RW Permission Check
          |
          v
Access Granted
Remember: Export Policy controls network-level access. UNIX permissions control file-level access. Both must allow access for a successful NFS operation.
