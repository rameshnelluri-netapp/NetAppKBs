NetApp BranchCache Tutorial

This tutorial explains what BranchCache is, how it works with NetApp ONTAP CIFS/SMB shares, the difference between Distributed Cache and Hosted Cache modes, and answers common interview questions.

1. What Problem Does BranchCache Solve?

Consider a NetApp CIFS share hosted in a central data center and users accessing it from a remote branch office. 

Branch Office Users
        |
        | WAN
        |
NetApp CIFS Share (Data Center)

Without BranchCache, every user downloads the same file across the WAN.

Example:

  • PowerPoint file size = 500 MB
  • 10 users open the file
  • Total WAN traffic = 5 GB
Repeated downloads consume WAN bandwidth and increase file access times.

2. What is BranchCache?

BranchCache is a Microsoft WAN optimization technology.

It allows Windows clients in branch offices to cache file content locally, reducing WAN traffic and improving access speed.

The original file always remains on the NetApp CIFS share. BranchCache only caches content locally.

3. What Does "ONTAP Acts as a BranchCache-Enabled SMB Content Server" Mean?

This statement often causes confusion.

ONTAP does NOT store the branch cache.

ONTAP's role is:

  • Store the original file
  • Generate content hashes
  • Provide hashes to Windows clients
  • Allow clients to validate cached data

The actual cache resides on:

  • Windows clients (Distributed Cache mode)
  • Windows Hosted Cache server (Hosted Cache mode)
NetApp provides the original content and validation hashes. Windows performs the actual caching.

4. File Access Without BranchCache

User1 ---> WAN ---> NetApp CIFS Share
User2 ---> WAN ---> NetApp CIFS Share
User3 ---> WAN ---> NetApp CIFS Share

Every user downloads the same file from the storage system.

5. Distributed Cache Mode

Step 1 - First User Opens File

PC1 ---> WAN ---> NetApp CIFS Share
  • PC1 downloads the file
  • PC1 stores BranchCache data locally

Step 2 - Second User Opens Same File

PC2 first checks whether the content exists locally in the branch office. 

PC2 --> "Who has this content?"
PC1 --> "I do"

PC2 retrieves cached blocks from PC1. 

PC2 <---- LAN ---- PC1

Instead of: 

PC2 ---- WAN ----> NetApp
WAN traffic is avoided and access is significantly faster.

6. Important Question: Is User2 Accessing User1's File?

Not exactly.

BranchCache does NOT expose User1's files through SMB sharing.

Instead:

  1. NetApp provides content hashes
  2. Clients advertise cached content availability
  3. Clients exchange cached blocks
  4. Received blocks are validated using hashes

This is controlled cache sharing, not direct file sharing.

7. Security and Permissions

BranchCache never bypasses SMB permissions.

Example:

User Permission on CIFS Share Can Access Cached Content?
User1 Yes Yes
User2 Yes Yes
User3 No No
Even if content exists in the cache, users still require permission on the NetApp CIFS share.

8. Hosted Cache Mode

Instead of storing content on user machines, a dedicated Windows server stores cached content. 

PC1
PC2
PC3
  |
Hosted Cache Server
  |
  +---- WAN ----> NetApp CIFS Share

Clients retrieve cached content from the Hosted Cache server.

Benefits

  • Centralized cache management
  • Better scalability
  • Improved cache availability
  • Suitable for larger branch offices

9. Distributed Cache vs Hosted Cache

Feature Distributed Cache Hosted Cache
Dedicated Server Required No Yes
Cache Stored On User PCs Hosted Cache Server
Suitable For Small Offices Large Offices
Management Complexity Low Higher

10. NetApp ONTAP Commands

Check BranchCache Configuration

vserver cifs branchcache show

Enable BranchCache

vserver cifs branchcache create \
-vserver svm1 \
-hash-store-path /branchcache

Show CIFS Shares

vserver cifs share show

Enable BranchCache on a Share

vserver cifs share properties add \
-vserver svm1 \
-share-name DATA \
-share-properties branchcache

11. Windows Verification Commands

netsh branchcache show status all

Useful for checking:

  • BranchCache status
  • Cache mode
  • Cache size
  • Hosted Cache configuration

13. Interview Questions

Q1. What is BranchCache?

BranchCache is a Microsoft WAN optimization technology that allows Windows clients to cache SMB content locally and reduce WAN traffic.

Q2. Does NetApp Store the Cache?

No. NetApp stores the original file and provides content hashes. The cache resides on Windows clients or Hosted Cache servers.

Q3. Can User2 Access Cached Content from User1?

Yes, in Distributed Cache mode, clients can exchange cached blocks locally. However, SMB permissions are still enforced.

Q4. What Does ONTAP Acting as a BranchCache Content Server Mean?

ONTAP provides the original SMB content and generates hashes that Windows clients use to validate cached content.

14. Key Takeaways

  • BranchCache reduces WAN bandwidth usage.
  • Original files remain on NetApp CIFS shares.
  • ONTAP provides hashes, not cache storage.
  • Distributed Cache uses peer-to-peer client caching.
  • Hosted Cache uses a dedicated Windows server.
  • Security permissions remain enforced.
